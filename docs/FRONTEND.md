## 1. What the frontend is

A multi-vault, role-aware dashboard for the `my_project` Anchor
program at id `B7EUo8ipi5xNuTtjbrG6enXymac1bD4b6NijYAEFB45z`. Three
audiences, one app:

| Audience      | What they do                                                                                                |
| ------------- | ----------------------------------------------------------------------------------------------------------- |
| **User**      | Browse vaults, deposit / withdraw, inspect aggregate position + allocation pie                              |
| **Admin**     | Create strategies, set per-strategy weight, rotate the delegate (AI agent keypair), permanently deactivate  |
| **Authority** | Manual `allocate_to_strategy` / `deallocate_from_strategy` / `rebalance_strategy` / `report_yield`          |

Every vault is qualified by `(token_mint, vault_id)` — the same id on
two mints is a different vault. The active vault is encoded in the
URL as `/vault/[address]` (where `[address]` is the vault PDA), with
nested admin routes at `/vault/[address]/admin` and per-strategy
admin views at `/vault/[address]/admin/strategy/[id]`. The same
`VaultProvider` context still holds the active vault but is now
hydrated from URL params.

## 2. Stack (actually used)

| Layer            | Choice                                                                                                |
| ---------------- | ----------------------------------------------------------------------------------------------------- |
| Framework        | **Next.js 16.2.1** App Router, **TypeScript 6**                                                       |
| Wallet           | **`@solana/wallet-adapter-react` 0.15** + `@solana/wallet-adapter-react-ui` 0.9 + `@solana/wallet-adapter-wallets` 0.19 |
| Chain client     | **`@solana/web3.js` 1.98.4** + **`@coral-xyz/anchor` 0.32.1** + **`@solana/spl-token` 0.4.14**          |
| UI               | **Tailwind 4** + custom CSS variables (no shadcn). Radix is **not** used (the EVM playbook used it).   |
| Charts           | **Recharts 3** (donut for allocation)                                                                  |
| Toasts           | **react-hot-toast** wired in [app/src/app/layout.tsx](app/src/app/layout.tsx) and surfaced through [app/src/components/shared/TxToast.tsx](app/src/components/shared/TxToast.tsx) |
| ABI              | **Anchor IDL** — `target/idl/my_project.json` (regenerated by `anchor build`) is hand-mirrored to [app/src/idl/my_project.ts](app/src/idl/my_project.ts) |
| RPC              | Direct connection via `@solana/web3.js` to whatever `NEXT_PUBLIC_RPC_URL` points at — **no server-side proxy**     |

Notable pins / quirks:

- `@coral-xyz/anchor@0.32.1` matches the program toolchain pinned in
  [Anchor.toml](Anchor.toml) and [Cargo.toml](Cargo.toml). Bumping
  the program also requires a frontend bump.
- `next@^16.2.1` + `react@^19.2.4` + `typescript@^6.0.2` — relatively
  new versions; mind the React 19 strict-mode behavior in client
  components.
- The Next.js root layout defers state by using a **client provider
  tree** (`SolanaProvider` → `VaultProvider`), avoiding SSR
  hydration mismatches with the wallet adapter (see
  [app/src/app/layout.tsx](app/src/app/layout.tsx)).

## 4. Routes

URL-encoded vault selection via `[address]` (the vault PDA, base58)
is shipped. Static params for every entry in `VAULT_REGISTRY` are
generated at build time via `generateStaticParams`.

### `/` — multi-vault landing

[`page.tsx`](app/src/app/page.tsx) renders:

1. **`VaultList`** — clickable rows for every entry in
   `VAULT_REGISTRY`. Selecting a row navigates to
   `/vault/[address]`.
2. **`AggregateStats`** — cross-vault TVL / share-count aggregates.

### `/vault/[address]` — per-vault user dashboard

[`/vault/[address]/page.tsx`](app/src/app/vault/[address]/page.tsx)
renders:

1. **Vault header** — name + token symbol pill + paused banner if
   `vault_state.paused`.
2. **`VaultStats`** — asset symbol, TVL (= `total_deposited`), share
   price (computed via [format.ts:formatSharePrice](app/src/lib/format.ts)),
   strategy count.
3. **Two-column grid** (`lg:grid-cols-2`): left = a deposit/withdraw
   tab toggle wrapping `DepositForm` / `WithdrawForm`; right =
   `UserPosition`.
4. **`AllocationChart`** — Recharts donut, full-width below the grid.
5. **`ActivityFeed`** — decoded `#[event]` log stream for the active
   vault.

### `/vault/[address]/admin` — admin/authority surface

[`/vault/[address]/admin/page.tsx`](app/src/app/vault/[address]/admin/page.tsx)
wraps everything in [`AdminGuard`](app/src/components/admin/AdminGuard.tsx).
Inside the guard:

- [`CreateStrategyForm`](app/src/components/admin/CreateStrategyForm.tsx) — paste a delegate pubkey + an anti-theft checklist (UI only) → call `createStrategy(delegate)`.
- [`PauseToggle`](app/src/components/admin/PauseToggle.tsx) — admin-only `set_paused` switch.
- [`PerformanceFeeEditor`](app/src/components/admin/PerformanceFeeEditor.tsx) — `set_performance_fee_bps` within `[0, MAX_PERFORMANCE_FEE_BPS]`.
- [`AdminTransferFlow`](app/src/components/admin/AdminTransferFlow.tsx) — two-step `propose_admin` / `accept_admin` (and authority equivalents).
- [`AllowedTokensPanel`](app/src/components/admin/AllowedTokensPanel.tsx) — protocol-level `AllowedToken` PDA add / remove.
- A "Rebalance all" button → loops over active strategies and calls `rebalanceStrategy` for each.
- [`StrategyList`](app/src/components/admin/StrategyList.tsx) — grid of [`StrategyCard`](app/src/components/admin/StrategyCard.tsx) entries, each exposing per-strategy weight, delegate rotation, deactivation, and manual allocate / deallocate.

### `/vault/[address]/admin/strategy/[id]` — per-strategy admin

Drill-down from a `StrategyCard`: per-strategy `AllowedAction`
add/remove, delegate rotation, weight, deactivate.

## 6. RPC + cluster selection

No proxy. `@solana/web3.js`'s `Connection` is constructed by
[`SolanaProvider`](app/src/components/providers/SolanaProvider.tsx)
from `getRpcUrl()` (defaults to the cluster's public RPC). Cluster is
resolved from `NEXT_PUBLIC_CLUSTER` (`devnet` | `mainnet-beta`,
defaults to `devnet`). Override with `NEXT_PUBLIC_RPC_URL` to point
at a private RPC (Helius / Triton / a personal validator).

Caveats:

- Public devnet RPC throttles `getProgramAccounts`; the strategy fan-out
  query in [`useStrategies`](app/src/hooks/useStrategies.ts) will
  be slow on cold start. A private RPC is recommended for demo
  recordings.
- `NEXT_PUBLIC_TOKEN_MINT` overrides the default asset mint without
  editing the registry — useful for testing on a fresh devnet token.

## 7. Public feature inventory

### Wallet + navigation

- [`Navbar`](app/src/components/layout/Navbar.tsx) — sticky top
  bar with the Erebor brand, the active-vault selector, the
  `WalletMultiButton`, and (for admins/authorities) a link to
  `/admin`.
- [`VaultSelector`](app/src/components/layout/VaultSelector.tsx) —
  popover dropdown of every entry in `VAULT_REGISTRY`. Highlights
  the active vault from `VaultProvider`.

### Deposit

[`DepositForm`](app/src/components/vault/DepositForm.tsx). Flow
(via [`useDeposit`](app/src/hooks/useDeposit.ts)):

1. User enters an amount; `parseTokenInput` validates against
   `tokenDecimals`.
2. The **Max** button on the embedded
   [`AmountInput`](app/src/components/shared/AmountInput.tsx) fills
   the wallet token balance.
3. `program.methods.deposit(amount).accountsStrict({…}).rpc()` builds
   and sends. `vault_authority` signs the `mint_to`; the user signs
   the token transfer in.
4. On success, surface a toast and refetch
   [`useUserPosition`](app/src/hooks/useUserPosition.ts) +
   [`useStrategies`](app/src/hooks/useStrategies.ts).

There is **no separate `approve` step** — Solana doesn't use the
ERC-20 allowance pattern; the token transfer is signed inline by the
user.

### Withdraw

[`WithdrawForm`](app/src/components/vault/WithdrawForm.tsx). Mirror
flow via [`useWithdraw`](app/src/hooks/useWithdraw.ts) — input is
*shares to burn*, the program computes the corresponding underlying.

⚠ If the reserve can't cover the withdrawal, the call reverts with
`InsufficientReserve` (the program does not auto-pull from
strategies). The form surfaces the error message. See
[MISMATCHES.md §2.8](MISMATCHES.md).

### Allocation pie

[`AllocationChart`](app/src/components/admin/AllocationChart.tsx) —
Recharts donut with one slice for the reserve plus one per active
strategy. Center overlay shows TVL; legend underneath shows per-slice
percentages.

### User position card

[`UserPosition`](app/src/components/vault/UserPosition.tsx) —
shares balance + estimated underlying value (computed locally from
`shares * total_deposited / share_supply`). Refreshes on a polling
interval.

### Vault stats

[`VaultStats`](app/src/components/vault/VaultStats.tsx) — asset
symbol, TVL, share price (`formatSharePrice`), strategy count.

### Activity feed

✅ Shipped at [app/src/components/vault/ActivityFeed.tsx](app/src/components/vault/ActivityFeed.tsx).
Decodes Anchor `#[event]` records (added in Phase 2) via the program's
`BorshEventCoder`. Bootstraps from `getSignaturesForAddress` +
`getTransaction` for the most recent ~25 program transactions, then
subscribes to `connection.onLogs` for live updates. Filters to events
that reference the active vault PDA. Caps the visible list at 20 rows.
Allowed-action / value-source events will appear automatically once
those features ship.

### Pause UI

✅ Shipped. [PausedBanner.tsx](app/src/components/vault/PausedBanner.tsx)
shows a destructive banner across `/` and `/admin` whenever
`vault_state.paused` is true (plus an explanation that withdrawals
stay open). The admin-only [PauseToggle.tsx](app/src/components/admin/PauseToggle.tsx)
exposes a single button to flip the flag (disable-not-hide via
[useRoles.ts](app/src/hooks/useRoles.ts) — non-admins see the control
but it's `disabled`).

### Sum-cap weight warning

✅ Shipped in [StrategyList.tsx](app/src/components/admin/StrategyList.tsx).
Running progress bar above the cards shows the sum of `target_weight_bps`
across active strategies (the program enforces the cap as well via
`vault_state.total_active_weight_bps`, reverting with
`WeightSumExceedsMax`); shows the reserve buffer % when under-allocated.

## 8. Admin / authority feature inventory

### Create strategy

[`CreateStrategyForm`](app/src/components/admin/CreateStrategyForm.tsx) →
`createStrategy(delegate)` via
[`useAdminActions`](app/src/hooks/useAdminActions.ts). Validates
the delegate pubkey on input; the form also surfaces a manual
"checklist" (review code + audit, track record, key security) — UI
only, not enforced on-chain.

### Per-strategy card

[`StrategyCard`](app/src/components/admin/StrategyCard.tsx)
exposes:

- **Weight slider** — `setStrategyWeight(weight_bps)`. Steps of 100,
  capped at 10 000 per strategy. The contract enforces the sum cap
  across active strategies via `vault_state.total_active_weight_bps`
  (reverts with `WeightSumExceedsMax`); the UI mirrors this with a
  running sum-cap progress bar.
- **Update delegate** — `updateStrategyDelegate(new_delegate)`.
- **Deactivate** — `deactivateStrategy()`. Permanent.
- **Allocate / deallocate** — manual amounts via
  [`useAuthorityActions`](app/src/hooks/useAuthorityActions.ts).

### Strategy list

[`StrategyList`](app/src/components/admin/StrategyList.tsx) — grid
of cards plus the rebalance-all action.

### Authority manual rebalance

A **`rebalanceAll`** button in
[`/vault/[address]/admin/page.tsx`](app/src/app/vault/[address]/admin/page.tsx)
loops over active strategies and calls `rebalanceStrategy(id)` for
each. Phase-3 made `rebalance_strategy` authority-only at the program
level; the UI is weight-driven (target = `total_deposited * weight_bps
/ 10_000`). Spec ([§7.6](SOLANA_VAULT_SPEC.md)) additionally wants a
signed-delta push/pull radio — see [MISMATCHES.md §2.2](MISMATCHES.md).

### Yield reporting

`reportYield(strategy_id)` is wired in
[`useAuthorityActions`](app/src/hooks/useAuthorityActions.ts) but
not surfaced as a dedicated UI button today; it's invoked by
[`scripts/crank-yield.ts`](scripts/crank-yield.ts) /
[`scripts/simulate-yield.ts`](scripts/simulate-yield.ts) for the
demo.

### Allowed-action whitelist editor

✅ Shipped at the per-strategy admin route
(`/vault/[address]/admin/strategy/[id]`). Adds / removes
`AllowedAction` PDAs via [useAllowedActions.ts](app/src/hooks/useAllowedActions.ts).

### Allowed-token panel

✅ Shipped in [AllowedTokensPanel.tsx](app/src/components/admin/AllowedTokensPanel.tsx)
on the per-vault admin route. Manages the protocol-level `AllowedToken`
PDAs that gate which mints `initialize_vault` will accept.

### Config editor / value-source editor

❌ **Not built.** Blocked on the program adding the underlying
`AutoActionConfig` / `ValueSource` accounts. See
[MISMATCHES.md §2.1](MISMATCHES.md).

## 9. Role gating

[`AdminGuard`](app/src/components/admin/AdminGuard.tsx) reads the
on-chain `vault_state.admin` and `vault_state.authority` pubkeys
from `VaultProvider` and compares to the connected wallet's
`publicKey`. Role booleans (`isAdmin`, `isAuthority`,
`isPendingAdmin`, `isPendingAuthority`) are exposed via
[`useRoles`](app/src/hooks/useRoles.ts) and consumed by individual
controls to drive **disable-not-hide** UX (`PauseToggle`,
`PerformanceFeeEditor`, etc.). The route-level guard remains
all-or-nothing for the page itself; per-control disable is layered
inside admin pages.

## 11. Selectors and presets

`AllowedAction` editing is shipped, but the frontend does not yet
ship the EVM-playbook-style preset libraries (`ACTION_PRESETS` /
`KNOWN_ALLOWED_ACTION_TARGETS` / `signatureForSelector`). Today the
`(target_program, discriminator)` triple is entered manually.
`CONFIG_PRESETS` / `VALUE_SOURCE_PRESETS` will appear once the
corresponding program features (`AutoActionConfig`, `ValueSource`)
land (see [FRONTEND_PLAN.md](FRONTEND_PLAN.md) and
[MISMATCHES.md §2.1](MISMATCHES.md)).

   [app/src/idl/my_project.ts](app/src/idl/my_project.ts) if the
   IDL changed.
2. `bash scripts/deploy.sh` (only if redeploying the program).
3. `bun scripts/setup-devnet.ts` — full flow: mint test token, init
   vault, create N strategies, simulate yield. Or run the focused
   scripts ([create-vault.ts](scripts/create-vault.ts),
   [create-strategies.ts](scripts/create-strategies.ts),
   [simulate-yield.ts](scripts/simulate-yield.ts)).
4. `cd app && bun install && bun run dev`.
5. Multi-vault paths exercised:
   - [`VaultList`](app/src/components/vault/VaultList.tsx) renders
     every entry in `VAULT_REGISTRY`.
   - [`VaultSelector`](app/src/components/layout/VaultSelector.tsx)
     navigates between them without full reload.
6. Per-vault journeys:
   - **Admin:** create strategy, set weight, deactivate (verify
     permanence).
   - **User:** deposit, observe TVL increase + share price ≈ 1
     initially.
   - [`scripts/simulate-yield.ts`](scripts/simulate-yield.ts) →
     mint to strategy ATA → `report_yield` → share price ticks up →
     redeem and verify realized yield.
   - **Authority:** `allocate` / `deallocate` / `rebalanceAll` and
     confirm the pie updates.

## 14. Out of scope / deferred

- **Config / value-source editors** — blocked on the corresponding
  program features (`AutoActionConfig`, `ValueSource`)
  ([MISMATCHES.md §2.1](MISMATCHES.md)).
- **Action-preset libraries** (`ACTION_PRESETS`, target-program /
  discriminator catalogues) — `AllowedAction` editing is shipped,
  but pasting a `(target, disc)` pair is still manual.
- **Add custom vault dialog** — needs a paste-and-validate path
  against `program.account.vaultState.fetch(vaultPda)` plus
  `localStorage` persistence ([FRONTEND_PLAN.md](FRONTEND_PLAN.md)).
- **Server-side RPC proxy** — only matters if private RPC keys
  shouldn't reach the bundle.
- **Connected-wallet Playwright e2e** — wallet adapter's modal isn't
  headless-friendly; the unblock is a viem-style mocked connector
  using `Keypair.generate()` seeded with anvil-style funded accounts
  on a private validator.
- **Indexer-backed analytics** — running APY, drawdown charts,
  cumulative yield. Needs an indexer once a real vault has production
  history.

## 15. Porting notes

This frontend is a Solana port of an EVM dashboard playbook; the
following patterns are network-agnostic and worth carrying forward:

- **Vault is always `(cluster, token_mint, vault_id)` (Solana) or
  `(chainId, address)` (EVM).** Never trust one component alone.
- **Disable-not-hide for role-gated UI.** Adopted at the per-control
  level via `useRoles`; route-level guard remains all-or-nothing.
- **Build-time seed + runtime add via storage.** The
  `localStorage`-backed runtime-add pattern works the same on Solana
  (still a roadmap item).
- **Probe-by-known-pairs for non-enumerable on-chain state.**
  `AllowedAction` PDAs are probed by `(strategy, target_program,
  discriminator)` triple via
  `program.account.allowedAction.fetchNullable`.
- **Push/pull radio for signed deltas.** Don't make the user think
  in signed integers (roadmap).
- **Anti-theft enforcement is documented in the public popover.** The
  `(target_program, discriminator)` whitelist is on-chain via
  `AllowedAction` PDAs; surfacing it to non-admin users is a roadmap
  item.
